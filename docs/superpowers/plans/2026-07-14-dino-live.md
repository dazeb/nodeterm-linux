# Plan: live (shared) dino game ‚Äî spectator mode

**Branch:** `feat/dino-live` (worktree `/root/nodeterm-dino`, off `main`)
**Spec:** docs/superpowers/specs/2026-07-14-dino-live-design.md (approved)

## Summary

The dino game state is local-only, so a shared canvas shows independent dinos. Make it a **spectator
feature**: one authority plays, others watch the same game live ‚Äî by riding the presence layer (an
ephemeral `PeerState.dino` field, like `chat`). No new core channel; works over local/relay/browser
via the hub automatically.

## Tasks (subagent-driven, TDD, review each) ‚Äî sequential (one worktree)

### Task 1 ‚Äî Shared types + hub cast handler (foundation)
- `src/shared/presence.ts`: add `DinoSnapshot` (y, ducking, crashed, started, score, speed,
  groundScroll, obstacles[{kind,x,y,sx,sw,sh,flap}]); add `dino: { nodeId: string; snap: DinoSnapshot } | null`
  to `PeerState`; add caps `DINO_MAX_OBSTACLES` (‚âà 12) and a `sanitizeDinoPayload(payload): {nodeId,snap}|null`

- `src/core/presence/hub.ts`: handle a new `presence:dino` cast ‚Äî run `sanitizeDinoPayload`, set the
  sender's `PeerState.dino`, broadcast a `PeerDiff` `update`. Mirror the existing `presence:chat`

  so `dino` clears for free ‚Äî confirm no extra work.
- TDD (`hub.test.ts`): a `dino` cast sets the sender's `dino` + broadcasts an `update`; a payload with

### Task 3 ‚Äî dino-game engine: broadcast + spectate
- `src/renderer/nodes/dino/dino-game.ts`: add to `createDinoGame`'s opts an `onSnapshot?: (snap: DinoSnapshot | null) => void`
  and to its returned interface `setRemote(snap: DinoSnapshot | null): void` + `isAuthority(): boolean`.
  - **Broadcast:** in `frame()`, at `DINO_BROADCAST_HZ = 20` (throttle by wall-time since last emit),
    call `onSnapshot(currentSnapshot())` while the local sim is authoritative and `started || crashed`;
    call `onSnapshot(null)` when it returns to pre-start idle / on `stop()`/`destroy()`.
  - **Spectate:** `setRemote(snap)` with a non-null snap puts the game in REMOTE mode ‚Äî the rAF loop
    still runs (to draw) but `update()` is suspended and input (jump/duck) is ignored; `draw()` renders
    the remote `snap` (dino y/frame, obstacles, score, GAME OVER). `setRemote(null)` returns to LOCAL
    mode (own sim resumes from idle). Local input while in REMOTE mode ‚Üí immediately switch to LOCAL
    (take-over: seed the sim from the last remote snap for continuity), and the caller starts broadcasting.
- Keep the file self-contained (no imports beyond the DinoSnapshot type). Extract `currentSnapshot()`
  and the remote-draw as small internal functions.
- TDD (`dino/dino-game.test.ts`): drive the state machine without a real canvas where possible (mock
  the 2d context) ‚Äî `setRemote(snap)` suspends update + renders the snap; a local key in remote mode
  switches to local + fires the take-over; `onSnapshot` throttles to ~20 Hz and yields null on idle.
  If a full canvas mock is too heavy, extract the mode/throttle/snapshot logic into a tiny pure
  module and test THAT; state what you did.

### Task 4 ‚Äî DinoNode wiring (authority + spectate + label + tiebreak)
- `src/renderer/nodes/DinoNode.tsx`: it's under the active-session provider.
  - Get the active presence: `const presence = useActiveSessionPresence()` and `const { api } = useSession()`.
  - Pass `onSnapshot: (snap) => api.presence.dino(snap ? { nodeId: id, snap } : null)` to `createDinoGame`.
  - Subscribe `presence.store(useShallow((s) => selectDino(s, id)))` ‚Üí the peer (if any) broadcasting
    for THIS node. When present AND that peer should win the authority tiebreak (**lower clientId wins**;
    if OUR clientId is lower we keep local play and ignore the remote), call `game.setRemote(peer.dino.snap)`
    on each new snapshot; when absent, `game.setRemote(null)`.
  - Render a small "‚ñ∑ <name> is playing" label (peer color) while spectating.
  - On unmount, `api.presence.dino(null)` if we were the authority (belt-and-suspenders; the engine's
    `destroy()`/`stop()` already emits null via onSnapshot).
- TDD: extract any non-trivial decision (the tiebreak: given my clientId + the broadcasting peer, do I
  spectate or keep playing?) into a tiny pure helper + unit-test it. The JSX wiring relies on typecheck
  (no jsdom for DinoNode).
